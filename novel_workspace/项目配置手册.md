================================================================================
一、系统架构概述
================================================================================

这是一个"基于状态机的长篇小说自动化生成系统"。核心思路：

  ┌─────────────┐     ┌──────────────┐     ┌──────────────┐
  │ 00_Knowledge │────▶│ run_chapter  │────▶│ 02_Drafts    │
  │ 世界观/角色/ │     │ .ps1         │     │ chapter_N.md │
  │ 故事大纲     │     │ 组装提示词   │     │              │
  └─────────────┘     │ ↓ 调DeepSeek │     └──────────────┘
                       │ ↓ 保存文件   │
  ┌─────────────┐     └──────────────┘
  │ 01_State    │────▶
  │ JSON状态机  │     每生成一章后 → Agent手动覆写状态机 → 下一章基于新状态
  └─────────────┘

关键设计决策：
  - 为什么用 PowerShell 而不是 Python？
    因为本机没有安装 Python 解释器，但 PowerShell 5.1 自带的 .NET 类库
    足以完成 HTTP 请求和 UTF-8 文件读写。WebClient + UploadString 方案
    经过了多轮编码兼容性调试。

  - 为什么用状态机而不是直接把所有上下文塞进提示词？
    DeepSeek 的上下文窗口有限。状态机只保留最近5条事件和5条未解线索，
    保证每一章都能拿到"故事进行到哪了"的精确快照。

二、环境准备
================================================================================

【操作系统】
  Windows 10/11，PowerShell 5.1+

【唯一的"安装"操作】
  在 scripts\.env 文件中填入你的 DeepSeek API 密钥：

    DEEPSEEK_API_KEY=sk-your-actual-key-here

  不需要 pip install。不需要 npm install。不需要 Python。不需要 Node.js。

【API 端点】
  POST https://api.deepseek.com/v1/chat/completions
  Model: deepseek-chat

【验证密钥是否可用】
  PowerShell 终端中执行以下命令（替换 $key 为实际值）：

    $key = "sk-xxx"
    $wc = New-Object System.Net.WebClient
    $wc.Headers.Add("Authorization", "Bearer $key")
    $wc.Headers.Add("Content-Type", "application/json")
    $body = '{"model":"deepseek-chat","messages":[{"role":"user","content":"test"}],"stream":false}'
    $wc.UploadString("https://api.deepseek.com/v1/chat/completions", "POST", $body)

  若返回包含 "choices" 字段的 JSON，则密钥有效。

四、00_Knowledge —— 知识库配置
================================================================================

这三个文件是"静态数据"。它们只在切换世界观或新增核心角色时手动修改。

【world_setting.md 范例】（校园甜文版）
  # 世界观设定
  ## 时代背景
  现代社会，华国。城市全部以首字母为命名，如H市A市。都市校园。
  ## 社会结构
  符合现代当今社会的基本社会结构。
  ## 底层生态
  符合现代当今社会的基本社会特征。

【characters.md 结构】（角色设定的核心字段）

  ## 主角：许灼
  ## **身份**：高三一班人气担当，云初的青梅竹马。
  ## **年龄**：18。高大挺拔，阳光帅气。笑起来极具感染力。
  【核心原则】：阳光开朗，不是高冷男神，情商很高，非常会说话。
  **性格**：阳光开朗、高段位社交、极度双标。
  **核心动机**：对感情→"偏爱"和"服软"只对云初一人可见。

  每个角色都需要三个字段：
  1. 【核心原则】—— 不可违反的铁律，写在生成指令里
  2. **性格** —— 决定这个角色的说话方式和行为
  3. **核心动机** —— 决定这个角色在故事中的驱动力

【main_outline.md 结构】
  分四卷，每卷有"阶段定位 + 核心事件"。
  第一卷模板：
    第一卷：夏日沸点（1~5万字）
    阶段定位：高三冲刺期，日常高甜。
    核心事件：
    - 开篇定调
    - 天降试探（苏乔）
    - 假想敌出现（沈时砚）
    - 卷末高潮


================================================================================
五、01_State —— 状态机配置
================================================================================

【JSON 结构定义】

{
  "current_chapter": 2,            ← 脚本拿这个数字生成 chapter_NNNN.md
  "current_chapter_to_write": 3,   ← 下一章要写的编号
  "protagonist": {
    "name": "许灼",
    "physical_state": "描述主角当前身体/心理状态",
    "equipment": ["物品1", "物品2"],  ← 每章更新，增删物品
    "location": "当前所在地点"
  },
  "recent_events": [               ← 最近5条剧情进展
    "事件1的简短概括",
    "事件2"
  ],
  "unresolved_threads": [          ← 当前未解决的悬念
    "线索1",
    "线索2"
  ]
}

【状态更新时的关键规则】

规则1：recent_events 绝对不能预写"下一章才会发生的事"
    BAD:  "周六去云初家吃饺子" ← 如果第三章还没生成就写在状态机里
           → API 会认为该事件已发生 → 跳过描写
    GOOD: 只写第二章实际发生的事："许灼在全班面前摊牌→云初回应前十条件"

规则2：unresolved_threads 删除已解决的线索，新增新产生的悬念
    第二章前："沈时砚即将与云初同坐讨论联考" ← 待解决
    第二章后："沈时砚已消解（去了办公室而非教室讨论）" ← 从线程中删除

规则3：equipment 随故事推进增删
    第一章："睡衣/T恤/一柱擎天的下身"
    第二章："物理错题本"新增 ← 情感道具
    第三章："草莓软糖只剩最后两颗" ← 反映消耗


================================================================================
六、scripts —— 脚本与提示词配置
================================================================================

【.env ← 一行就够了】
  DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

【system_prompt.txt ← 只用写一次，不动】
  你是一位专业的小说作者。
  请严格遵循以下创作准则：
  1. 禁用一切AI陈词滥调，包括但不限于"仿佛在诉说""命运的齿轮""心中涌起一股暖流"等。
  2. 用具体的感官细节（视觉、听觉、触觉、嗅觉）和动作来暗示情绪，绝对不要直接描述情绪。
  3. 对话必须口语化，不同角色要有明显不同的说话习惯和语气词，禁止用对话交代背景信息。
  4. 严格符合人物设定，不允许OOC。
  5. 允许句式不完美、有瑕疵，追求真实的手感，像人类讲述，不要过于工整。
  6. 本文是都市校园小甜文，尽量写的甜蜜一些。

【director_notes.txt ← 每章重写一次】
  这是控制生成质量最关键的入口。
  每章开始前，手动编写节拍式指令。好的指令要包含：
    1. 所有场景的清单 → API会不会漏掉节拍取决于这里是否明确
    2. 指定对话 → 如果关键对白不想被替换，必须写死
    3. 人设重申 → "云初不是高冷女神""许灼话多爱笑"等每章都要复读
    4. 不许出场的人 → "本章不允许转校生苏乔出现"
    5. 字数要求 → "4000-5500字"

【run_chapter.ps1 ← 核心执行脚本，不需要改】
  脚本做了什么：
    ① 读 .env → 获取 $apiKey
    ② 读 system_prompt.txt → 获取 $systemPrompt
    ③ 读 director_notes.txt（可选） → 获取 $directorNotes
    ④ 读 world_setting.md、characters.md、main_outline.md → 组装上下文
    ⑤ 读 current_status.json → 拆出 $ch / $name / $phys / $loc / $events / $threads
    ⑥ 将所有数据拼成一个 userPrompt
    ⑦ 用 WebClient 向 DeepSeek API 发 POST 请求
    ⑧ 接收响应 → 提取 content → 保存为 02_Drafts/chapter_NNNN.md

  为什么用 WebClient 而不是 Invoke-RestMethod？
    PowerShell 5.1 的 Invoke-RestMethod 在处理 JSON 响应中的中文时，
    编码行为不稳定。System.Net.WebClient 配合 UTF8Encoding(false)
    （不带BOM）可以保证中文不出乱码。

七、执行流程
================================================================================

【一次完整的"写一章"流程】

Step 1：打开 director_notes.txt → 写入本章的大纲/指令/节拍
Step 2：打开 current_status.json → 确认 current_chapter 是正确的编号
        （第一次写的时候 current_chapter=1, current_chapter_to_write=2）
Step 3：PowerShell 终端执行：
          cd f:\SOLO\novel_workspace\scripts
          & ".\run_chapter.ps1"
Step 4：等待3-10秒 → 终端显示 "Done: ...chapter_NNNN.md"
Step 5：打开生成的文件审阅 → 如果节拍遗漏/人设崩坏 → 修改 director_notes.txt → 重新 Step 3
Step 6：定稿后 → 手动更新 current_status.json
Step 7：current_chapter_to_write 递增 → 回到 Step 1，开始下一章

【不需要手动做的事】
  - 不需要手动拼接提示词（脚本做）
  - 不需要手动调 API（脚本做）
  - 不需要手动命名文件（脚本按 chapter_NNNN.md 自动命名）


================================================================================
八、常见问题与Debug记录
================================================================================

【Bug 1】生成的内容没有任何中文，全是???或乱码
  原因：PowerShell 终端编码设置问题导致读取显示异常。
  解决：文件本身 UTF-8 内容是正确的，直接在 VS Code / 记事本里打开文件查看。
  终极方案：脚本用 [System.Text.UTF8Encoding]::new($false)（不带BOM的UTF-8）
  配合 WriteAllText 写入，保证文件编码一致。

【Bug 2】API 返回的内容被截断或中文大量丢失
  原因：Invoke-RestMethod 或 Out-File 的参数不兼容。
  解决：改用 System.Net.WebClient 的 UploadString 方法。

【Bug 3】API 跳过导演指令里的关键场景
  原因：状态机的 recent_events 预写了该场景。
  示例：第三章指令里要求写"周六去云初家吃饺子"，
  但 current_status.json 的 recent_events 里已经有了
  "周六去云初家吃饺子..." → API读到后认为已发生 → 跳过。
  解决：确保状态机中的 recent_events 只包含"已发生"的上一章事件。

【Bug 4】角色人设崩了
  原因：角色设定只在 characters.md 里写了一行，导演指令没复读。
  解决：导演指令里每次都要复述关键人设："许灼话多爱笑、从不反驳、
  喜欢光明正大"——三句话，不费事，但能解决90%的OOC。

【Bug 5】两段之间存在逻辑矛盾
  示例：第一章说许灼和云初是青梅竹马，第三章在房间看七岁照片像第一次见。
  原因：AI没有能力回溯前三章的所有细节做一致性校验。
  解决：生成后人工审读，发现矛盾手动修正。这是AI辅助写作的固有局限。

附录A：PowerShell 脚本完整代码（run_chapter.ps1）
================================================================================

$ErrorActionPreference = "Stop"
$utf8NoBom = New-Object System.Text.UTF8Encoding $false
$baseDir = $PSScriptRoot

$envContent = [System.IO.File]::ReadAllText("$baseDir\.env", $utf8NoBom)
$apiKey = ""
if ($envContent -match 'DEEPSEEK_API_KEY=(.+)') {
    $apiKey = $matches[1].Trim()
}

$systemPrompt = [System.IO.File]::ReadAllText("$baseDir\system_prompt.txt", $utf8NoBom)
$directorNotes = ""
$directorPath = "$baseDir\director_notes.txt"
if (Test-Path $directorPath) {
    $directorNotes = [System.IO.File]::ReadAllText($directorPath, $utf8NoBom)
}

$world = [System.IO.File]::ReadAllText("$baseDir\..\00_Knowledge\world_setting.md", $utf8NoBom)
$chars = [System.IO.File]::ReadAllText("$baseDir\..\00_Knowledge\characters.md", $utf8NoBom)
$outline = [System.IO.File]::ReadAllText("$baseDir\..\00_Knowledge\main_outline.md", $utf8NoBom)
$stateJson = [System.IO.File]::ReadAllText("$baseDir\..\01_State\current_status.json", $utf8NoBom)
$state = $stateJson | ConvertFrom-Json

$ch = $state.current_chapter
$name = $state.protagonist.name
$phys = $state.protagonist.physical_state
$loc = $state.protagonist.location

$recentLines = @()
foreach ($e in $state.recent_events) { $recentLines += "  - $e" }
$recentStr = $recentLines -join "`n"

$threadLines = @()
foreach ($t in $state.unresolved_threads) { $threadLines += "  - $t" }
$threadStr = $threadLines -join "`n"

$directorBlock = ""
if ($directorNotes -ne "") {
    $directorBlock = @"



$directorNotes
"@
}

$userPrompt = @"

$world


$chars


$outline


current_chapter: $ch
name: $name
physical_state: $phys
location: $loc
recent_events:
$recentStr
unresolved_threads:
$threadStr


$($stateJson)
$directorBlock
"@

$body = @{
    model = "deepseek-chat"
    messages = @(
        @{ role = "system"; content = $systemPrompt },
        @{ role = "user"; content = $userPrompt }
    )
    stream = $false
} | ConvertTo-Json -Depth 6

$wc = New-Object System.Net.WebClient
$wc.Encoding = $utf8NoBom
$wc.Headers.Add("Authorization", "Bearer $apiKey")
$wc.Headers.Add("Content-Type", "application/json")
$responseJson = $wc.UploadString("https://api.deepseek.com/v1/chat/completions", "POST", $body)
$responseObj = $responseJson | ConvertFrom-Json
$content = $responseObj.choices[0].message.content

$draftDir = "$baseDir\..\02_Drafts"
if (-not (Test-Path $draftDir)) { New-Item -ItemType Directory -Force -Path $draftDir | Out-Null }
$chapterNum = $ch.ToString("0000")
$outPath = "$draftDir\chapter_$chapterNum.md"
$content | Out-File -FilePath $outPath -Encoding utf8

Write-Host "Done: $outPath"
