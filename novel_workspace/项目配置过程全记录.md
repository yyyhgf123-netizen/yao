阶段3：脚本调试与平台迁移
================================================================================

【3.1 环境问题】
  系统没有安装 Python、Node.js、winget。
  原本的 generate_chapter.py 无法运行。

【3.2 转向 PowerShell 直连方案】
  Agent 创建 run_chapter.ps1，用 PowerShell 原生能力：
    - [System.IO.File]::ReadAllText 读取所有知识库文件
    - ConvertFrom-Json 解析状态机
    - Invoke-RestMethod 调用 DeepSeek API
    - Out-File 保存结果

【3.3 编码噩梦（多次修复）】
  Bug 1：单行命令中文被吞
    → 改为独立 .ps1 脚本文件
  Bug 2：here-string @" 解析错误
    → 改用数组 -join
  Bug 3：Write 工具写入 .ps1 后中文变乱码
    → 把中文提示词拆分到 system_prompt.txt
  Bug 4：Invoke-RestMethod 解码中文出错
    → 改用 System.Net.WebClient，设置 Encoding = UTF8NoBom
  Bug 5：Out-File -Encoding utf8BOM 参数不支持
    → 改为 -Encoding utf8
  Bug 6：Invoke-WebRequest NullReferenceException
    → 回退到 Invoke-RestMethod（编码由 WebClient 上层接管）

最终稳定方案：
  - WebClient.UploadString → 原始字节 UTF-8 读取 → ConvertFrom-Json → WriteAllText
